# f-read 隐私政策

最后更新：2026-01-26

## 1. 数据收集范围

- `chrome.storage.sync`：模型提供商/模型名、API Key（明文存储于浏览器配置中）、代理 URL（可选）、输出语言、快捷键、内置/自定义 Prompt 配置。
- `chrome.storage.local`：按网页 URL 聚合的会话历史（用户提问、AI 回复、锚点引用编号）、摘要快照的标题与 favicon、置顶/锁定标记。
- 运行时临时数据：当前标签页的 URL/title/favicon、抽取的正文文本、锚点定位信息、用户当前选中的文本（用于划词快捷问答）。
- 明确不收集：账号密码、浏览器 cookies、批量浏览历史、后台跟踪类数据；不做第三方行为分析与广告跟踪。

## 2. 数据使用目的

- 基于当前网页内容生成简洁摘要、全文总结、观点三要素，并支持锚点跳转。
- 支持多轮对话与划词快捷问答（仅在侧边栏开启时处理选中文本）。
- 持久化配置、Prompt 模板与历史会话，便于重复使用与回溯。

## 3. 传输与共享

- 不内置自有服务器，不向开发者上传任何用户数据。
- 当用户发起对话或快捷问答时，仅将必要的文本与用户配置的 API Key/代理 URL 一同发送到用户选定的 LLM 提供商（如 OpenAI、DeepSeek、Anthropic、Gemini）；传输依赖提供商的 HTTPS/安全通道。
- 不接入第三方分析/监控 SDK，不共享或出售任何数据。

## 4. 权限说明

- `<all_urls>`：在用户主动触发时对当前访问的网页执行正文抽取、锚点生成与高亮跳转；未知用户目标站点，需要匹配所有 URL。
- `activeTab` / `tabs`：仅在用户点击扩展或侧边栏开启时读取当前标签的 URL/title/favicon，并注入 content-script；不对后台标签批量扫描。
- `storage`：存储配置（sync）与本地历史（local），便于持久化。
- `sidePanel`：在 Chrome 侧边栏展示扩展 UI。

## 5. 数据存储与保留

- 所有数据仅存储在用户浏览器的 `chrome.storage.sync` 与 `chrome.storage.local`；不做云端备份。
- 用户可在 Options 页面删除 API Key/代理 URL、重置 Prompt，或清空历史会话；卸载扩展会移除所有本地数据。
- 若用户使用代理 URL，数据将按用户配置的代理路径传输，安全与合规由用户自担。

## 6. 安全与合规

- 不注入页面外的跟踪脚本，不拦截或修改用户的网络请求。
- 仅在需要时读取当前页内容，不在后台驻留收集。
- 建议用户为 API Key 设置最小权限，并定期更换；避免在不可信网络环境下填写密钥或代理。

## 7. 变更通知

- 隐私政策更新时将随扩展版本更新一同发布，并在仓库文档中同步更新日期。
